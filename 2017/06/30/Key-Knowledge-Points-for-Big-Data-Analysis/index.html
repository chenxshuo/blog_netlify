4. The definition , the main type, the characters and the risks of Cloud Computing

Answer:

Cloud computing is a style of computing in which dynamically scalable and often virtual resources are provided as a service over the Internet.

Cloud computing is a combination of distributed computing, utility computing, network storage, virtualization, load balance, high available and other advanced technologies.

The characters include:

(a)Super large-scale. (b) Virtualization. (c) High reliable. (d) Generality. (e) Extendibility (f)Service on Demand. (g) Extremely cheap.

Potential risks.

a) Public Cloud: Computing infrastructure is hosted at the vendor’s premises.

b) Private Cloud: Computing architecture is dedicated to the customer and is not shared with other organizations.

c) Hybrid Cloud: Organizations host some critical, secure applications in private clouds. The not so critical applications are hosted in the public cloud.

d) Community Cloud: Cloud is built based on Common Demands and Common Resources Sharing of communities.

7. The meaning of GNU

GNU is an operating system and an extensive collection of computer software. GNU is composed wholly of free software .GNU is a recursive acronym of “GNU is not Unix”,chosen because GNU’s design is Unix-like, but differs from Unix by being free software and containing no Unix code

wiki

简单说下方便理解，Unix出来之后很多公司希望通过软件赚钱，不再免费，不再开放源代码，这让崇尚开放，自由的一众技术人员很不满，其中一个Richard Matthew Stallman博士雄心勃勃的提出了一项计划GNU，其名称是GNU is not Unix的缩写（这里起名用到了一个递归，就是自己调用自己），他目标创建一个自由的，免费的，开源的并且像Unix一样好用的系统。

GNU项目一开始进展顺利不过到后来系统的内核部分迟迟不能令人满意，后来芬兰的一个大学学生Linus受GNU项目的启发，开发了一个简易的操作系统Linux即（Linus’s Unix), 放到网上吸引了许多程序员参与维护修改更新，最终发展成为今天的Linux。

8. The meaning of GPL

General Public License is a widely used free software license which guarantees end users the freedom to run, study, share and modify the software.

wiki

GPL协议就是一个开放源代码协议，软件的初始开发者使用了GPL协议并公开软件的源程序后，后续使用该软件源程序开发软件者亦应当根据GPL协议把自己编写的源程序进行公开。GPL协议要求的关键在于开放源程序，但并不排斥软件作者向用户收费。虽然如此，很多大公司对GPL协议还是又爱又恨，爱的是这个协议项下的软件历经众多程序员千锤百炼的修改，已经非常成熟完善，恨的是必须开放自己后续的源程序，导致竞争对手也可以根据自己修改的源程序开发竞争产品。﻿﻿

The main structure is Master/Slave

Master mainly uses NameNode to manage the whole file system and provide access channels for customers. NameNode takes the charge of HDFS storage. Slaves mainly use Data node to Manage the data on its storage point.

20. The feature of three framework

Samza: Timely process streaming data, all data are separated into different partitions, and appened into a queue. Other distributed computational components can be deployed on Samza easily and flexibly, and handle the information timely.

Spark: Use time windows to get streaming data from a certain time period as RDD. Use transformation to make the RDD available to be read, write and etc. RDD is a data object of Spark, this feature mean that Spark can process large-scale dataset at one time, thus it is suitable to be used for machine learning tasks.

Storm: The distributed calculation capability is based on its topology network, in which spout is used for receiving data streaming and provide response; Bolt is used for receive data from spout and invoke functions to process data and return it back.

22. Describe the process of K-means algorithms

  • First , Describe the process of K-means algorithms

  1. Pick k cluster centroids randomly as $\mu_1, \mu_2,… \in R^n$

  2. Repeat the calculation below till convergence:

    for every single example i, calculate the class it belongs to , the way is :

    $c^{(i)} = argmin_j||x^{(i)} - \mu_j||^2$

    ​

    for every class called j, calculate the centroids again:

    $\mu_j = {\sum_{i=1}^m 1{c^{(i)} = j }x^{(i)}}/\sum_{i=1}^m 1 {c^{(i)} = j}$

    ​

  • Second use map-reduce to realize the process

    Step-1.Create a K length dict asdict{“1”:[], “2”:[], “3”:[], “4”:0 …… “K”:[]}

    Step-2. Random select K nodes as primary centroid. Like:

​ Dict{“1”: [0.3, 0.8, 0.5, 0.2,0.9 … ]

​ “2”: [0.5, 0.7, 0.6, 0.4, 0.7… ]

​ “3”: [0.5, 0.7, 0.6, 0.4, 0.7… ]

​ ……}

Step-3. Use Label-Id “\t” Score askey-value to get the score of each node id under each Label (First MapReduce).

Step-4. Use Id “\t” Label-Score askey-value to get the minimal one, assign the output as Id “\t” Label (SecondMapReduce).

Step-5. Use Id “\t” Label tore-calculate centroid and update Dict.

Step 6. Repeat Step 3 until converge.

26. The process description in Map and Reduce stages

Map Stages:
a. Get the input data from HDFS and Split it into certain amount of segments.
b. Send each segment to different mappers.
c. Mappers finish processing each segment and store the data back on HDFS.

Reduce Stages:
a. Use TaskTracker to get the Mapper output from HDFS, and send them to reducer.
b. Each reducer merge the mappers’ results, which are assigned to it.
c. All reducers’ results are combined together and sent back to HDFS.
