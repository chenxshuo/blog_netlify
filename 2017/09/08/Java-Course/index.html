1.1 字节

字节（Byte）——8个二进制位为1个字节

比特（bit）——二进制数字的位，信息量的最小单位

补充学习

  • 二进制、八进制、十进制、十六进制的换算方法
  • 计算机数的原码、反码、补码表示法等概念

1.2 软件工程

编写程序的过程称程序设计

编写程序使用的语言称程序设计语言

1.2.1 程序设计语言

计算机程序设计语言分为

  • 机器语言
  • 汇编语言
  • 高级语言

$$高级语言 \xrightarrow{编译连接} 机器语言目标程序 \xrightarrow{执行} 结果 $$

1.2.2 数据和算法

一个程序最基本的两部分内容：

  1. 数据结构

    对数据的描述，即程序中要指定的数据类型和数据的组织形式

  2. 算法

    对操作的描述，即程序的操作步骤，是程序设计的灵魂

1.2.3 软件工程和软件的定义

软件是程序、方法、规则、相关文档以及在计算机上运行所必需的数据的集合。

软件工程是开发、运行、维护软件的系统方法。

软件生命期指从开始研制到废弃不用的整个期间，可划分为五个阶段：

  • 需求分析
  • 设计
  • 编程
  • 测试
  • 运行维护

软件的质量标准：正确性 、 健壮性、 可维护性 、 可用性、 可重用性、 效率等

两种常讲的程序设计

  • 传统的结构化程序设计 SP(Structured Programming)
  • 面向对象的程序设计 OOP(Object Oriented Programming)

结构化程序设计为处理复杂问题提供了有力手段，但到80年代末，这种设计方法逐渐暴露出以下缺陷：

  1. 难以适应大型软件的设计。
  2. 程序可重用性差。

面向过程程序设计缺点的根源在于数据与数据处理分离。

面向对象程序设计模拟自然界认识和处理事物的方法，将数据和对数据的操作方法放在一起，形成一个相对独立的整体——对象（object），同类对象还可抽象出共性，形成类（class）。

一个类中的数据通常只能通过本类提供的方法进行处理，这些方法成为该类与外部的接口。对象之间通过消息（message）进行通讯。

第二章 Java语言概述

Java是当今最流行也是最有前途的面向对象的网络编程语言

2.1 Java历史概述

  • Java语言诞生于20世纪90年代初期。
  • Java语言的前身是SUN公司开发的一种用于智能化家电的名为Oak(橡树)的语言，它的基础是当时最为流行的C和C++语言。
  • 1993年，WWW（万维网）迅速发展，SUN公司发现可以利用Oak语言的技术来创造含有动态内容的WWW网页，便将改造后的Oak语言改名为Java语言（程序员手中的咖啡）。
  • 在1995年，Java被定位于网络应用程序设计语言。
  • Java Servlet的推出，Java在电子商务方面开始崭露头角，尤其是JSP技术的出现，而成了当前主流的网络编程语言。

2.2 Java平台运行环境

Java是解释执行的高级编程语言

Java解释执行的过程

$$Java源程序(.java文件) \xrightarrow{编译} 字节码程序(.class文件) \xrightarrow[执行]{解释} 解释器(Java虚拟机) \rightarrow 操作系统 \rightarrow 裸机 $$

其他编译型语言的执行过程

$$编译型源程序 \xrightarrow[连接]{编译} 可执行程序 \xrightarrow{执行} 操作系统 \rightarrow 裸机 $$

字节码是Java虚拟机（Java Virtual Machine-JVM）的指令组。字节码最大的好处是可跨平台运行，即Java的字节码可以编写一次，到处运行。

任何一种可以运行Java字节码的软件均可看成是Java的“虚拟机”，可以把Java的字节码看成是JVM上所运行的机器码(machine code) 。

2.3 Java虚拟机

  • Java 实现平台无关性，主要是靠Java虚拟机
  • Java虚拟机是一种利用软件方法实现的抽象化的计算机，有自己虚拟的处理器、堆栈、寄存器等硬件设备，还具有相应的指令系统。它是基于操作系统和硬件平台之上，可以在上面执行Java的字节码程序。
  • Java虚拟机将在内部创建一个运行时系统，帮助以下列方式执行代码：
    • 加载 .class文件
    • 管理内存
    • 执行垃圾收集

2.4 Java平台的版本

  • Standard Edition(标准版)

    桌面系统开发平台

  • Enterprise Edition(企业版)

    企业级应用平台，提供分布式企业软件组件架构

  • Micro Edition(嵌入式版)

    小型设备与技能卡的嵌入式平台

    ​借助这些平台版本，使得Java程序能在各式大小型机器及操作系统上运行，达成所谓程序可移植性、跨平台的目标。

2.5 Java语言特点

Java与C++

​ Java语言有选择地继承了C++语言的语法规则和面向对象机制，放弃了C++复杂，安全性差，不适合网络应用等特性。但在组织结构上却相当的不同。它是一个完全面向对象的程序语言，程序的基本处理单位：类（class）及API的组织，均以对象的观点来处理及架构起来的。

  • 简单
    • 类似于C++的风格，使得C++程序员非常容易掌握Java
    • 摒弃C++中容易引发程序错误的地方，如指针操作和内存管理
  • 面向对象
    • Java中万物皆对象。这使得编程重点集中于数据和数据操作方法
  • 分布式
    • 数据分布：数据可以分布在网络的不同主机上
    • 操作分布：一个计算可以分散在不同的主机上处理
  • 健壮性
    • Java在编译和运行时都对可能出现的问题进行检查
    • 提供自动的垃圾回收功能来进行内存管理
    • 面向对象的异常处理机制
  • 安全
    • Java语言提供的安全性
    • 编译器提供的安全性
    • 字节码校验器
    • 类加载器
  • 体系结构中立
    • Java程序被编译成与体系结构无关的字节代码
    • 只要安装了Java运行环境，Java程序可以在任意处理器上运行
  • 可移植性
    • 源代码级
    • 目标代码级
  • 解释执行
    • 编译器javac处理java源文件并生成类文件
    • 类文件通过java解释器进行解释执行，将Java字节码转换位机器可执行代码
  • 多线程
    • Java语言内置支持多线程的功能
    • Java类库提供了Thread类

2.6 Java程序的种类

​ Java 是一种网络程序语言，它能写出包含前端(Applet)、本地（Application）及后端（Servlets）的程序。而JSP(Java Server Pages)技术，是运行于Web服务器端的标记文件，把Java程序嵌入到html页面文档中，可以快速开发和部署动态网页的网络应用。

Applet已经成为了历史淘汰的产物

Servlets和JSP现在也已经很少使用了，但是作为Java网络框架的基础还是需要好好学习

2.7 Java Application 程序基本结构

2.7.1 应用程序命名规则

  1. 源文件的扩展名必须是 .java 。
  2. 如果源文件中有多个类，则最多只能有一个public类，如果有的话，那么源文件的名字必须与这个public类的名字相同；如果源文件没有public类，那么源文件的名字由用户任意命名。主类：main方法是程序执行的入口。

不适用IDE，即集成开发环境时，只是单纯的编辑Java源文件，推荐使用Visual Studio Code、Atom或notepad

2.7.2 第一个applet小程序

Java Applet 是另一类非常重要而特殊的Java程序，它本身不能单独运行，必须嵌入到html文件中，借助浏览器或Appletviewer来解释执行。Applet可以在网络上进行传输，由浏览器自动下载并执行，所以大大提高了Web的交互能力、动态执行能力和丰富Web的多彩画面。
