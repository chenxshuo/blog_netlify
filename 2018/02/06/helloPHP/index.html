Constants

A constant is a name or an identifier for a simple value. A constant value can’t change during the execution of the script. By convention, constants identifiers are always uppercase.

  • Only scalar data ( boolean, integer, float ,string )can be contained in constants.
  • There is no need to write $ before a constant.
  • Constants can only be defined using define()
  • Constants can be defined and used everywhere.
  • You can use constant() to read a value of a constant
  • A constant name starts with a letter or underscore.

An array is a data structure that stores one or more similar type of values in a single value.

  • Associative array

    An array with string as index. This stores element values in association with key values .

  • Multidimensional array

    An array containing one or more arrays and values are accessed using multiple indices

GET and POST Methods

There are two ways the browser client can send information to the web server:

  • GET method
  • POST method

GET

The GET method sends the encoded user info appended to the page request.

  • restricted to 1024 characters only
  • Never use GET if you have sensitive info
  • GET can’t be used to send binary data like image to the server
  • The data sent by GET can be accessed using QUERY_STRING variable
  • PHP provides $_GET associative array to access all the sent info

POST

POST method transfer info via HTTP headers.

  • Don’t have restriction on size
  • Can be used to send ASCII and binary data
  • By using Secure HTTP you can make info secure
  • $_POST

$_REQUEST

The $_REQUEST variable contains the content of both $_GET and $_POST and $_COOKIE .

File Inclusion

You can include the content of a PHP file into another PHP file. This will help developers to make it easy to change the layout of website with minimal effort.

  • include()

    Take all the text in a specified file and copies it into file that uses the include function. If there is a problem in loading a file, then the include() function will throw a warning but script will continue execution.

  • require()

    If there is a problem in loading a file, the require() function will generates a fatal error and halt the execution of the script.

Regular Expression

Regular Expression is nothing more than a sequence of pattern of characters.

  • POSIX Regular Expression
  • PERL Style Regular Expression

POSIX Regular Expression

Brackets [ ]

Brackets are used to find a range of characters.

  • [0-9], [1-3]
  • [a-z],[b-f]
  • [A-Z]
  • [a-Z]
Quantifiers: + * ? { } $
  • p+ : one or more p
  • p* : zero or more p
  • p? : zero or more p
  • p{N} : a sequence of N p
  • p{a,b} : a sequence of a or b p, like p{2,3}
  • p{a, } : a or more p
  • p$ : with p at the end of it
  • ^p : with p at the beginning of it
  • p.p : p + any character + p
Examples
  • [^a-zA-Z]
  • p.p
  • ^.{2}$
  • <b>(.*)<b>
  • p(q)*
Predefined Characters Ranges
  • [[:alpha:]] : a-Z
  • [[:digit:]] : 0-9
  • [[:alnum:]] : 0-9 a-Z
  • [[:space:]] : a white space

PERL Style Regular Expression

Meta-characters
  • . : a single char
  • \s : a whitespace character (space tab newline..)
  • \S : non-whitespace char
  • \d : a digit 0-9
  • \D : a non-digit
  • \w : a word char a-z, A-Z, 0-9, _
  • \W : a non-word char
  • : matches a single char in the given set
  • [^ ] : matches a single char outside the set
  • ( aa | bb | cc) : matches any of the alternative specified
Modifiers
  • i
  • m
  • o
  • s
  • x
  • g
  • cg
